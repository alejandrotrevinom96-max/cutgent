# Communication — Binary Rubric

Run before presenting or sending. Each line is yes/no. Unchecked boxes are your
revision list.

- [ ] The core message is written in ONE sentence.
- [ ] The audience is named, with what they know/want/fear.
- [ ] The goal is explicit: what they should think, feel, or do afterward.
- [ ] There is a deliberate opening hook (not "today I'll talk about…").
- [ ] There is clear tension/stakes before the resolution.
- [ ] The structure is intentional (story spine, pyramid, or hook-tension-resolution) — not a topic dump.
- [ ] At least one concrete example or story supports the main point.
- [ ] Jargon is removed or defined for THIS audience; clarity beats cleverness.
- [ ] Transitions are signposted and the message is recapped at the end.
- [ ] Each slide carries at most one idea (no bullet walls; detail moved to appendix/handout).
- [ ] The talk has been rehearsed aloud, on your feet, and timed to fit.
- [ ] Filler words are minimized and the key line is marked with a pause.
- [ ] The likely hard questions are anticipated with honest answers prepared.
- [ ] There is one clear call to action / ask.
- [ ] Every section earns its place by serving the one core message (anything that doesn't is cut).
