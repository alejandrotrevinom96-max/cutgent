# Rubric — Business Strategy (BINARY)

Each line is pass/fail; check only if verifiably true with evidence or stated
assumptions. Personal context can override a default, but a defensible answer is
still required for *this* business.

## Problem & customer
- [ ] A specific target customer (ICP) is named — segment, role, context — not "everyone."
- [ ] The Job To Be Done / core problem is stated in one sentence.
- [ ] Problem severity is validated with evidence (interviews, behavior, willingness to pay), not assumed.
- [ ] The current alternative (including "do nothing") is identified.

## Value proposition
- [ ] The value proposition states who/problem/why-better in one sentence.
- [ ] The offering beats the status-quo alternative by a clear, stated margin.

## Business model & economics
- [ ] Price and revenue model are explicit.
- [ ] Gross/contribution margin is computed or estimated with stated assumptions.
- [ ] CAC and LTV (or payback period) are estimated; LTV/CAC and payback are stated.
- [ ] Unit economics are positive OR a credible, specific path to positive is written.

## Moat & positioning
- [ ] A specific moat source is named (scale, network effects, switching costs, brand, cornered resource, counter-positioning, cost/process).
- [ ] Positioning vs competitors and substitutes is articulated (not "we execute better").

## Go-to-market
- [ ] The GTM motion matches the buyer and price point.
- [ ] A beachhead segment and expansion path are identified.

## Metrics & prioritization
- [ ] Exactly ONE north-star metric tied to delivered customer value is chosen.
- [ ] No vanity metrics are used as primary success measures.
- [ ] The top 1–3 priorities are sequenced against the current biggest risk/constraint.

## Honesty
- [ ] Assumptions are labeled and separated from validated facts.
- [ ] The strongest counter-argument to the strategy is stated.
