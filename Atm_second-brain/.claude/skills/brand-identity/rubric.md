# Brand Identity Rubric (binary — every box must be checked before an identity ships)

- [ ] Positioning is written as one explicit sentence (audience, category, differentiated promise, reason to believe) before any visual was made.
- [ ] The target audience is named specifically, including what the brand must signal about *them*.
- [ ] Brand personality is 3–5 adjectives, each paired with a rejected opposite ("warm, not cute").
- [ ] Differentiation is explicit: category convention is named, and where this brand diverges is stated on purpose.
- [ ] Every major visual/verbal decision traces to a strategy attribute with a written *because*.
- [ ] A logo SYSTEM exists (primary, secondary/stacked, monogram/icon, favicon-scale) — not a single lockup.
- [ ] The logo passes the small-size test (legible at ~16px favicon) AND scales up cleanly.
- [ ] The logo passes the monochrome test (solid black, knockout white, single ink) without relying on color/gradient.
- [ ] Logo clearspace, minimum sizes, and misuse rules are defined in measurable terms.
- [ ] A type system is defined (≤2 families or one superfamily), with a hierarchy/scale and confirmed web/app licensing + fallback stack.
- [ ] A color system defines primary, secondary, and functional roles as named tokens — not a mood image.
- [ ] Every text/background color pair meets WCAG AA contrast, and no meaning is carried by hue alone.
- [ ] Verbal identity defines a constant voice AND a tone range that flexes by context, plus a lexicon (use/avoid words).
- [ ] At least one distinctive brand asset (color/shape/character/sound/device) is defined and checked for fame + uniqueness.
- [ ] The identity is pressure-tested across multiple real touchpoints (incl. dark mode / app icon / social avatar) and flexes without losing recognition.
- [ ] No element clones a competitor's equity, and the identity makes no misleading claim about origin or credentials.
- [ ] `personal/` recall has been consulted; any conflict with generic advice is flagged and personal/ wins.
