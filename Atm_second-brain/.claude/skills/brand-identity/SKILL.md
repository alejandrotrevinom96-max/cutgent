# Brand identity

A brand identity is a SYSTEM, not a logo. The job is to make the brand instantly
recognizable, consistent everywhere, and ownable. Strategy decides everything;
visuals only execute it. Generic best practice is the floor. **Whenever `personal/`
recall returns the user's real brand — their positioning, audience, existing
palette/marks, voice notes, prior decisions — that OVERRIDES the generic moves
here.** Surface the conflict, then follow personal/.

## 1. Strategy first (no pixels yet)

Refuse to design until these are written sentences, not vibes:

- **Positioning.** One sentence: for [audience], [brand] is the [category] that
  [single differentiated promise], because [reason to believe].
- **Audience.** Who specifically, what they already feel/believe, what they need
  the brand to signal about *them* when they use it.
- **Personality.** 3–5 adjectives, each with an explicit opposite you reject
  ("warm, not cute"; "precise, not cold"). Adjectives with no rejected opposite
  are decoration.
- **Differentiation.** What competitors all look/sound like — then deliberately
  diverge. Category convention is the baseline you break on purpose.

## 2. Derive attributes → design decisions

Every visual/verbal choice must trace to a strategy attribute. "Bold + modern"
→ high-contrast type, confident scale, restrained palette. Write the *because*
next to each decision; a choice with no traceable why is arbitrary and will drift.

## 3. Logo system (not one lockup)

- **Responsive marks:** primary lockup, secondary/stacked, monogram/icon, and a
  favicon-scale mark. One logo that must work everywhere fails somewhere.
- **Clearspace + min sizes** defined in measurable units.
- **Scalability test:** legible at favicon (16px) AND on a billboard.
- **Monochrome test:** works in solid black, solid white (knockout), and one ink.
  If it dies without color or gradient, it's not a logo, it's an illustration.
- **Misuse rules:** what must never be done (stretch, recolor, re-effect, crowd).

## 4. Type system (not fonts)

- **Pairing:** typically a display/brand face + a workhorse text face; max two
  families unless one superfamily covers all weights.
- **Hierarchy:** a defined type scale (sizes, weights, line-height, tracking) so
  any layout is composable, not bespoke.
- **Performance + licensing:** confirm web/app licensing and weights; subset and
  self-host or use webfont service; have a system-font fallback stack.

## 5. Color system (not a palette photo)

- **Primary** (the ownable brand color), **secondary** (support/accent),
  **functional** (success/warning/error, neutrals/ink). Define each as a token.
- **Accessibility:** every text/background pair meets WCAG AA contrast; never
  carry meaning by hue alone.
- **Meaning + context:** account for color associations and cultural context in
  the brand's markets; specify values across media (digital, print, environment).

## 6. Verbal identity

- **Voice** = the constant personality in words. **Tone** = how it flexes by
  context (error message vs launch vs apology) — define the range, not one setting.
- **Lexicon:** words/phrases the brand uses and ones it bans; naming conventions;
  grammar/mechanics rules. Verbal identity is as ownable as the logo.

## 7. Consistency = tokens + guidelines

- Encode color/type/space as **named tokens** (one source of truth) so the same
  system drives web, app, deck, print, and social.
- Guidelines show **do AND don't**, give ready assets, and explain the *why* so
  others extend the system correctly instead of copying surface.

## 8. Application & flexibility

Pressure-test across real touchpoints (favicon, app icon, business card, social
avatar, billboard, packaging, swag, dark mode). A rigid system breaks at the
edges; a coherent one *flexes without losing recognition*.

## Distinctiveness & memory

Build **distinctive brand assets** (Romaniuk/Ehrenberg-Bass): a color, shape,
character, sound, or device the audience links to you with no logo present. Test
each on *fame* (how many link it to the brand) and *uniqueness* (how few link it
to anyone else). Recognition over cleverness; consistency over novelty.

## Honesty boundary

This pack builds *your* ownable identity. It will not clone another brand's
equity (look-alike marks, copied palettes/assets), produce misleading identity
(impersonation, fake origin/credentials), or trade craft for trend-chasing. A
strong identity is distinct and true, not borrowed.

## Pull the personal layer

The user's positioning, audience, existing marks/palette/voice, and past
decisions live in their notes. Pull them via `recall`; the pack supplies the
system, the personal layer supplies *theirs*. QC output against `rubric.md`.
