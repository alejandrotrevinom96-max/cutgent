# Decision-Making Exemplars

## One-way vs two-way doors (Bezos)
Type 1 (irreversible — deliberate) vs Type 2 (reversible — decide fast, walk back if
wrong). The org failure is applying slow Type-1 process to fast Type-2 decisions.
**Borrow:** classify the door first; if cheaply undoable, decide now.

## Resulting (Duke, *Thinking in Bets*)
Evaluating a decision by whether it worked out. A great call can lose to variance.
**Borrow:** review the process and probabilities you used, not just the result; don't
punish a sound bet that lost.

## Premortem — prospective hindsight (Klein)
Assume it's a year later and the project failed; have everyone write why. Framing
failure as certain surfaces more candid risks. **Borrow:** run a 5-minute "it bombed —
what happened" before any big call.

## Inversion and mental models (Munger)
"All I want to know is where I'm going to die, so I'll never go there." List how you'd
guarantee failure, then avoid those. Use a latticework of models. **Borrow:** ask
"how would this surely fail?" and design those paths out.

## Base rates and System 1/2 (Kahneman)
People anchor on the vivid inside-view story and neglect base rates. **Borrow:** before
trusting your estimate, ask how the reference class usually turns out and anchor there.

## Superforecasting — incremental updating (Tetlock)
The best forecasters break questions down, start from base rates, assign explicit
probabilities, update in small steps, and keep score. **Borrow:** attach a number and
confidence, update gently as news arrives, review calibration afterward.
