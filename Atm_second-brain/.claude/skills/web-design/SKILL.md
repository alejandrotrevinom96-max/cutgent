# Web design

Award-level ≠ decorative. It means ruthless hierarchy, deliberate space, and
craft in the details. Depth lives in the companion files; load them as needed.

## Method

1. **Goal and one job per screen.** What is the single action or message? Design
   everything to serve it; cut what doesn't.
2. **Hierarchy first, prettiness later.** Establish the visual order (what the eye
   hits 1st/2nd/3rd) with size, weight, and space before choosing colors.
3. **Type system, not fonts.** Pick a scale (e.g. modular), set line-length
   45–75ch, line-height ~1.5 body. Two families max.
4. **Space is the design.** Use a spacing scale (4/8px base). Generous, *uneven*
   whitespace that groups related things beats uniform padding.
5. **Color with intent.** One accent that earns attention; the rest a restrained
   neutral range. Check contrast (WCAG AA minimum).
6. **Motion as meaning.** Animate to explain state/relationship, 150–300ms,
   eased; never decoration that delays the user.
7. **Responsive by content.** Design the small screen and the large screen, not
   just a desktop that shrinks.
8. **QC against `rubric.md`.** Pull the user's `personal/` brand notes via
   `recall` — their system overrides generic best practice.

## Pull the personal layer

Brand palette, voice, prior decisions, and component conventions live in the
user's notes. The pack supplies craft; the personal layer supplies *theirs*.
