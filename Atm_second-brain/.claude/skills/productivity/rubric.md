# Rubric — productivity system (binary)

Check a box only if literally true right now. Where the user's `personal/` context
sets a different bar, theirs wins — note it.

## Capture and clarify
- [ ] A single trusted capture inbox exists and is actually used for new inputs.
- [ ] Inboxes are processed to zero on a regular cadence.
- [ ] Every task is written as a concrete next physical action.
- [ ] Multi-step outcomes are tracked as projects, each with a defined next action.
- [ ] Date-specific items live on the calendar; non-dated tasks do not clutter it.

## Organization
- [ ] Projects/areas/resources/archive (PARA) are distinguished, using the vault's structure.
- [ ] A waiting-for / delegated list exists and is reviewed.

## Prioritization
- [ ] Priorities are set by leverage/importance, not by whatever is loudest or latest.
- [ ] Each day has 1–3 high-leverage tasks identified before reactive work begins.
- [ ] Urgent-but-unimportant items are batched, delegated, or declined — not auto-done.

## Focus and execution
- [ ] At least one distraction-free deep-work block is scheduled and defended.
- [ ] Work-in-progress is limited (you finish before starting more).
- [ ] Single-tasking is the default; notifications are off during focus blocks.
- [ ] The hardest cognitive work is placed in your peak-energy window.

## Boundaries and maintenance
- [ ] You have declined or descoped at least one low-leverage commitment recently.
- [ ] A weekly review is scheduled and was actually done this week.
- [ ] Recovery/downtime is protected, not treated as slack to be filled.
