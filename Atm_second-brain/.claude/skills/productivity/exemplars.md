# Exemplars — canonical productivity workflows

## 1. GTD (David Allen)
Capture → Clarify → Organize → Reflect → Engage. Capture every open loop; clarify
each (actionable? next action? <2 min do now, else defer/delegate); organize by
list (next-actions, projects, waiting-for, someday/maybe, calendar for dated only);
the **weekly review**; engage by context/time/energy/priority. **Example:** "Plan
Mom's birthday" is a *project*; its next action is "text siblings to pick a date."

## 2. Deep Work (Cal Newport)
Distraction-free concentration produces the high-value output; shallow work
(email, logistics) is batched and minimized. Schedule deep-work blocks as
appointments; pick a depth ritual; track deep-work hours. **Example:** 9–11 daily,
door closed, phone elsewhere, one document; email only after 11 in a batch.

## 3. Eisenhower matrix
Important+Urgent → do now. Important+Not-urgent → *schedule* (deep work, planning,
health — where leverage lives and most under-invest). Not-important+Urgent →
delegate/batch. Not-important+Not-urgent → delete.

## 4. Timeboxing
Assign every hour to a block in advance; rebuild when reality intervenes. Converts
an infinite to-do list into a finite, finishable day. **Example:** a calendar (9–11
deep work, 11–11:30 email, 1–2 meeting, 2–3:30 second deep block) instead of a
14-item list.

## 5. PARA (Tiago Forte) — tie to the vault
Organize all information by actionability: Projects, Areas, Resources, Archive.
Use the vault's existing PARA folders so tasks and knowledge live in one structure
— don't spin up a separate task app that drifts from the vault.

## 6. The weekly review (keystone)
~30–60 min, same time weekly: empty every inbox to zero; review the calendar (past
loose ends, next two weeks' prep); ensure every active project has a next action;
prune someday/waiting-for; check projects against goals. By the end your head is
empty and you start the week trusting the system.
