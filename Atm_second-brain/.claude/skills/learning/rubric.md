# Rubric — learning project (binary)

Check a box only if literally true right now. Where the user's `personal/` context
sets a different bar, theirs wins — note it.

## Definition and decomposition
- [ ] The target skill is stated as a concrete thing you will be able to *do*.
- [ ] Success criteria are written and measurable (a test of "done").
- [ ] A real deadline or checkpoint exists.
- [ ] The skill is broken into named sub-skills.
- [ ] The current bottleneck sub-skill is identified and prioritized.

## Practice and feedback
- [ ] Practice targets the edge of ability (you fail some of the time).
- [ ] A concrete feedback source exists (answer key, tests, coach, engine, rubric).
- [ ] Feedback arrives quickly and you act on the specific errors.
- [ ] Practice is direct — done in the context the skill is actually used.

## Memory mechanics
- [ ] Active recall is used (self-testing), not rereading or highlighting.
- [ ] A spaced review schedule with expanding intervals is set.
- [ ] Problem types / topics are interleaved, not blocked.

## Understanding and encoding
- [ ] You have explained the material from scratch (Feynman) and found the gaps.
- [ ] New material is elaborated and linked to prior knowledge / own examples.
- [ ] Learnings are encoded as atomic evergreen notes in the vault, in your own words.
- [ ] Testable atoms are turned into SRS cards made from material you already understand.

## Honesty checks
- [ ] Difficulty is present — nothing here feels comfortable or passive.
- [ ] You can demonstrate the skill, not just recognize the material.
