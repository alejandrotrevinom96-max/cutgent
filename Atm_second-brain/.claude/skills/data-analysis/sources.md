# Sources — Data Analysis

The personal layer (the user's data, metric definitions, domain, tooling)
overrides generic best practice. Date-stamp fast-moving tool/library docs.

- **John W. Tukey** — *Exploratory Data Analysis* (1977). Look before you test.
- **Edward R. Tufte** — *The Visual Display of Quantitative Information* (2001).
  Data-ink, chartjunk, small multiples, graphical integrity.
- **Cole Nussbaumer Knaflic** — *Storytelling with Data* (2015). One message per
  chart; declutter; narrative for decisions.
- **Alex Reinhart** — *Statistics Done Wrong* (2015). p-hacking, multiple
  comparisons, base rates, power.
- **Gelman & Hill** — *Data Analysis Using Regression and Multilevel Models* (2007);
  "garden of forking paths." (See also *Regression and Other Stories*, 2020.)
- **Larry Wasserman** — *All of Statistics* (2004). Estimation, testing, bootstrap.
- **Anscombe** (1973) and **Matejka & Fitzmaurice** (2017, Datasaurus). Plot it.
- **Judea Pearl** — *The Book of Why* (2018). Causal reasoning; correlation != causation.
- **Kohavi, Tang & Xu** — *Trustworthy Online Controlled Experiments* (2020). A/B
  testing: power, peeking, guardrails.

## How to weigh them
Prefer the simplest analysis that answers the decision honestly. When in doubt,
plot the data and report an interval. The user's metric definitions win.
